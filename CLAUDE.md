# CLAUDE.md

Guidance for Claude Code (and other AI agents) working in this repository. Keep this file accurate —
update it when conventions change, and record durable lessons in the **Lessons learned** section below.

## Project

`@ngui/auto-complete` — an Angular autocomplete that works both as a **directive** (`[ngui-auto-complete]`,
attached to any input) and as a standalone **component** (`<ngui-auto-complete>`).

Angular CLI monorepo with two projects:

- `projects/auto-complete` — the published library (this is what ships to npm).
- `projects/demo` — the demo app, built to `docs/` and deployed to GitHub Pages by CI.

The demo **consumes the built library from `dist/`**, not the source — so you must rebuild the library
before the demo (or your editor) sees library changes.

## Versioning

The library's major version tracks the supported Angular major (`@ngui/auto-complete@N` ⇒ Angular `N`).
Don't introduce a major bump that doesn't line up with an Angular major.

## Commands (run from the repo root)

```bash
npm install
npm run build-lib:prod   # build the library to dist/ (run this BEFORE build-docs / before testing the demo)
npm run build-docs       # production build of the demo app into docs/
npm run lint             # ng lint (ESLint flat config, eslint.config.js)
npm test                 # ng test (or per project, headless, below)
npx ng test auto-complete --watch=false --browsers=ChromeHeadless
npx ng test demo --watch=false --browsers=ChromeHeadless
npm run format           # prettier --write
npm run format:check     # prettier --check (CI gate)
npm run cypress:run      # e2e
```

## Pre-PR validation gate

Before opening any PR for a code change, all of these must pass:

1. `npm run build-lib:prod`
2. `npm run build-docs` (watch for `NG8113` unused-import warnings)
3. `npm run lint`
4. `npx ng test auto-complete --watch=false --browsers=ChromeHeadless`
5. `npx ng test demo --watch=false --browsers=ChromeHeadless`
6. `npm run format:check`

## Workflow & conventions

- **Small, single-concern PRs.** One topic per branch/PR — it keeps review and the changelog clean.
- **Branch from `master`**, open a PR, and **stop** — let a maintainer review and merge. Don't self-merge.
- **Conventional Commits.** Breaking changes use `!` in the title and a `BREAKING CHANGE:` footer.
- **Docs go with the code:** update `CHANGELOG.md` for every user-facing change, `README.md` when the
  public API/usage changes, and add a `MIGRATION.md` section for breaking changes.
- **Prettier is enforced** (`.prettierrc.json`: printWidth 140, tabs, single quotes, trailing commas) via
  a husky + lint-staged pre-commit hook. Markdown is in `.prettierignore` (don't expect it to be formatted).
- **Never commit a generated build timestamp:** `projects/demo/src/app/build-info.ts` is a committed
  placeholder regenerated by the `prebuild`/`prestart` hooks — revert it before committing.
- Editor/template diagnostics for the demo can be **stale until you rebuild the library** — trust the
  `build-docs` result over the IDE.

## Library specifics (gotchas)

- **Inputs are signal `input()`s** (read-only). In unit tests, set them with
  `fixture.componentRef.setInput('<public-alias>', value)`, not `component.x = …`. The directive forwards
  inputs to the dynamically created dropdown component via `ComponentRef.setInput()` for the same reason.
- **Public input names keep their kebab-case aliases** (e.g. `min-chars`, `list-formatter`). Don't rename
  them for aesthetics — it breaks every consumer template for no functional gain.
- **`output()` defaults to `<void>`** — use `output<T>()` for events that carry a payload.
- Numeric inputs use `numberAttribute`, booleans use `booleanAttribute`, so both `min-chars="2"` and
  `[min-chars]="2"` are accepted and correctly typed.
- **Angular 21 runs zoneless by default.** If a test mutates a child component's fields out of band, call
  `markForCheck()` on that child's `ChangeDetectorRef` before `detectChanges()` to avoid a false `NG0100`
  (`ExpressionChangedAfterItHasBeenChecked`).
- On `ng update`, pin **all** `@angular/*` packages to the same major, and check the `package.json` diff
  for silently-narrowed `eslint`/`typescript-eslint` ranges.

## Lessons learned

- The demo builds against `dist/`, so library API changes aren't visible to the demo (or accurate in the
  IDE) until `npm run build-lib:prod` runs.
- Signal `input()`s are read-only: tests and the directive set them via `ComponentRef.setInput()`.
- `output()` with no type parameter is `<void>`; pass `output<T>()` (or `output<any>()`) for payloads.
- Angular 21 zoneless change detection surfaces out-of-band child-field mutations as `NG0100` — mark the
  view for check first (see above).
